# ProductCamp Referral Bot

Telegram-бот на `grammy` для поддержки реферальной программы ProductCamp. Позволяет генерировать персональные реферальные ссылки, отправлять медиа и приглашения, создавать фирменные аватарки, показывать результаты и рассылать сообщения администраторам.

## Возможности

- `/start` — стартовое сообщение, в том числе с параметром `/start avatar`
- `/help` — помощь и описание функций
- `/results` — финальные результаты реферальной программы
- генерация реферальной ссылки
- генерация аватарки на основе Telegram-аватара
- показ шаблонов для сторис
- генерация QR-кода на реферальную ссылку
- отправка приглашений из Google Sheets
- админская рассылка по списку `chat_id`

## Установка

1. Склонируйте репозиторий:

```bash
git clone <repo-url>
cd pc_referral_bot
```

2. Установите зависимости:

```bash
npm install
```

3. Создайте файл `.env` на основе `.env.example` и заполните значения.

## Переменные окружения

Пример переменных уже есть в `.env.example`.

- `TG_TOKEN` — токен Telegram-бота
- `G_SHEET_ID` — ID Google Spreadsheet
- `FIREBASE_CONFIG` — JSON-конфигурация Firebase
- `G_API_CONFIG` — JSON-конфигурация Google Service Account
- `GA_CONFIG` — JSON-конфигурация Google Analytics / Measurement API
- `ADMINS` — список admin username через запятую
- `TZ` — часовой пояс, например `Europe/Moscow`

> Важно: JSON-внутри `.env` должен быть корректным и экранированным.

## Запуск

```bash
npm run build
npm start
```

Для локальной разработки можно использовать `ts-node`, если он установлен отдельно.

## Скрипты npm

- `npm run build` — компиляция TypeScript
- `npm start` — компиляция + запуск `node dist/bot.js`
- `npm run start:prod` — запуск через `pm2`
- `npm run start:debug` — запуск откомпилированного бота в debug режиме
- `npm run restart` — перезапуск всех процессов `pm2`
- `npm run restart:build` — пересборка и перезапуск
- `npm run stop` — остановка всех процессов `pm2`
- `npm run clean` — удаление директории `dist`

## Команды бота

### Пользовательские

- `/start` — приветствие и основное меню
- `/help` — справка по возможностям
- `/results` — показать результаты кампании

### Callback-кнопки

- `generatorMoreData` — дополнительная информация о продвижении
- `promoMaterials` — материалы для продвижения
- `getStories` — отправка шаблонов для сторис
- `generateAvatar` — показать варианты аватарок
- `avatar-center`, `avatar-round`, `avatar-left`, `avatar-right` — выбрать вариант аватарки

### Встроенные сценарии

- Генерация реферальной ссылки
- Отправка QR-кода для реферальной ссылки
- Сбор и отправка приглашения из листа `invites`

### Админские команды

- `start_broadcast` — запуск рассылки через разговор `brodcastMessage`
- `admin_panel` — админская панель

> Админские функции доступны только пользователям, имя которых содержится в `ADMINS`.

## Архитектура

- `bot.ts` — инициализация бота и подключение middleware
- `core/router` — кастомный роутер для команд и callback-событий
- `core/commands.ts` — бизнес-логика команд
- `core/services` — сервисы для Google Sheets, QR, аватарок и ссылок
- `core/middlewares` — расширение контекста, аналитика, логирование
- `assets/` — изображения для аватарок и сторис

## Настройка Google Sheets

В проекте используется `google-spreadsheet` и сервисный аккаунт Google.

- В `G_API_CONFIG` передается JSON-ключ сервисного аккаунта
- Таблица должна содержать листы:
  - `links` — сохранение данных реферальных ссылок
  - `logs` — список `chat_id` для рассылок
  - `invites` — приглашения для генерации текста

## Примечания

- Локализация сейчас поддерживает русский и англоязычную ветку, но основная логика ориентирована на русский контент
- Поддержка аватарок строится на получении аватара Telegram и наложении маски через `sharp`
- Если в `.env` отсутствует JSON, то конфиг загрузится безопасно и выведет предупреждения
