# Контекст проекта: ProductCamp Referral Bot

## Общее описание

Telegram-бот на `grammy`, созданный для поддержки реферальной программы ProductCamp. Основные функции включают генерацию персональных реферальных ссылок, создание фирменных аватарок, отправку медиа и приглашений, демонстрацию результатов и возможность админской рассылки.

## Структура проекта

-   `bot.ts`: Главный файл инициализации бота, подключение middleware.
-   `core/`:
    -   `commands.service.ts`: Сервис, управляющий логикой команд.
    -   `commands.ts`: Основные обработчики команд и сценариев.
    -   `config.ts`: Конфигурация и работа с переменными окружения.
    -   `router/`:
        -   `routes.ts`: Определение маршрутов для команд и callback-событий.
        -   `routes.enum.ts`: Перечисление всех возможных маршрутов.
    -   `services/`:
        -   `link.service.ts`: Работа с реферальными ссылками и Google Sheets.
        -   `qr-code.service.ts`: Генерация QR-кодов.
        -   `assets.service.ts`: Обработка аватарок и медиа.
        -   `broadcast.service.ts`: Рассылка сообщений.
        -   `google_api/`: Сервисы для взаимодействия с Google Sheets, Analytics и аутентификации.
-   `assets/`: Статические файлы (изображения для аватарок и сторис).

## Основные команды бота

-   **Пользовательские команды**:
    -   `/start` — Приветствие и основное меню.
    -   `/help` — Справочная информация.
    -   `/results` — Показ результатов кампании.
-   **Callback-кнопки**: Позволяют пользователю взаимодействовать с меню (например, `getStories`, `generateAvatar`).
-   **Админские команды**:
    -   Доступны только пользователям из списка `ADMINS` в `.env`.
    -   Включают запуск рассылки (`start_broadcast`) и доступ к админ-панели (`admin_panel`).

## Настройка и запуск

1.  **Переменные окружения**: Настройка через `.env` файл (на основе `.env.example`). Ключевые переменные:
    -   `TG_TOKEN`: Токен бота от BotFather.
    -   `G_SHEET_ID`: ID Google Таблицы.
    -   `ADMINS`: Список username админов через запятую.
2.  **Запуск**:
    -   `npm run build` — Сборка проекта.
    -   `npm start` — Запуск бота.

## Разработка

-   **Работа с кодом**: Все изменения в кодовую базу вносятся через Pull Request.
-   **Ветвление**: Новые ветки создаются с префиксом `pcr-{number}` (например, `pcr-01-fix-typo`).
-   **Ревью кода**: Перед слиянием в основную ветку (`main`) обязательна проверка кода.
